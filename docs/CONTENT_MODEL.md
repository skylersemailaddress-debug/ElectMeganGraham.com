# Content Model (Phase 1)

## Purpose
Separate content from layout so updates are faster and safer.

## Files

### homepage.json
- hero (headline, lead, CTAs)
- trust pills
- priority blocks

### about.json
- intro
- biography (to be filled)
- why running
- qualifications

## Rules
- Do NOT edit content directly inside HTML once wired
- Update JSON files instead
- Keep structure stable

## Expansion (future)
- events.json
- voter-tools.json
- endorsements.json

## Workflow
1. Edit JSON
2. Commit to GitHub
3. Site auto updates
